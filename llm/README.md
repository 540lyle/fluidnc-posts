# LLM Context

This folder contains compressed context for coding agents and future maintainers.

## Read order

1. `repo-map.md`
2. `architecture.md`
3. `backlog.md` for planning, roadmap, or documentation tasks
4. `../docs/phase-5-roadmap.md` if the task touches next capability slices or fixture expansion
5. `fusion-adapter.md` or `testing.md`, depending on task
6. `legal-and-upstream.md` if the task touches publishing, licensing, or upstream baselines

## Usage rule

Prefer these files over rereading the whole repository. They exist to keep context small and decisions consistent.

If the task is about next steps, planning, or scope expansion, read `backlog.md` and `../docs/phase-5-roadmap.md` before the domain-specific summaries.
